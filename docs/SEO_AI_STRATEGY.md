## 3. Traditional SEO Strategy

### 3a. Technical SEO (infrastructure layer)

The infrastructure layer is complete. Key facts:

- **Sitemap** lives at `nextjs-app/app/sitemap.ts` (Next.js auto-serves at `/sitemap.xml`). It covers: static routes (home, portfolio, posts, photography, resume, about), plus all `portfolioProject`, `post`, and `photoPost` documents fetched via the `sitemapData` GROQ query.
- **Robots.txt** lives at `nextjs-app/app/robots.ts`. Disallows `/api/` and `/studio/`, points to sitemap.
- **metadataBase** is set in `app/layout.tsx` `generateMetadata()`. Falls back to `process.env.NEXT_PUBLIC_SITE_URL` then `https://maxcsh.vercel.app`. Never leaves metadataBase undefined.
- **Canonical URLs** are auto-generated by Next.js from metadataBase â€” no manual `<link rel="canonical">` needed.

**Environment variable:** `NEXT_PUBLIC_SITE_URL` must be set in Vercel to the production URL. If unset, the hardcoded fallback is used. Every `SITE_URL` constant in page files follows the same pattern:

```ts
const SITE_URL = process.env.NEXT_PUBLIC_SITE_URL || "https://maxcsh.vercel.app";
```

### 3b. On-Page Metadata (per-route rules)

**Rule:** Every route must export `title` + `description` metadata.

| Route | Type | Title source | Description source |
|-------|------|-------------|--------------------|
| `/` (root) | `generateMetadata` async | Sanity settings.title | Sanity settings.description |
| `/posts/[slug]` | `generateMetadata` async | post.title | post.excerpt |
| `/portfolio/[slug]` | `generateMetadata` async | project.title | project.shortDescription |
| `/photography/[slug]` | `generateMetadata` async | post.caption (first 80 chars) | post.caption |
| `/resume` | Layout wrapper metadata | "Resume" (static) | 120â€“160 char static description |
| List pages (/portfolio, /posts, /photography) | Static export | Descriptive static string | Static description |

**Title format:** `%s | Max Chen â€” Portfolio` â€” template is set once in root layout, inherited by all pages.

**Description length:** 120â€“160 characters, written for humans. Avoid keyword stuffing.

**"use client" pages** cannot export `metadata` directly. Use a `layout.tsx` wrapper in the same directory that exports `metadata`, then keep the page component as a client component. Example: `app/resume/layout.tsx`.

### 3c. Open Graph + Twitter Cards (social sharing layer)

**Applied to:**

The global layout sets `openGraph.images` as the site-level OG image fallback. Detail pages override with their own image via `previousImages` spread pattern.

### 3d. JSON-LD Structured Data (rich results layer)

JSON-LD determines eligibility for Google rich results and provides the primary signal for AI knowledge graph accuracy.

**Injection pattern** (add before visible markup in the page component return):

```tsx

  dangerouslySetInnerHTML={{ __html: JSON.stringify(jsonLd) }}
/>
```

**Schema map by route:**

## 4. AI Search Optimization Strategy

AI-powered search engines (Perplexity, ChatGPT Browse, Bing AI, Claude) read pages differently from Google. They synthesize content to answer questions, not just rank pages. Optimize for being cited accurately, not just found.

### 4a. llms.txt Standard

`/llms.txt` (served from `nextjs-app/public/llms.txt`) is an emerging plain-text convention that tells LLM crawlers who owns the site and what it contains. Analogous to `humans.txt` but for AI agents.

**Keep this file updated** when Max's role, employer, or major skills change. It is the most direct signal available to AI crawlers for building an accurate author profile.

Format: plain prose + structured URL index. No HTML. No markdown. Simple, crawlable.

### 4b. Semantic HTML & Content Clarity

AI agents parse semantic markup more reliably than CSS class names. Standards to maintain:

- `<article>` wraps blog post and project page body content âś… (done in posts/[slug])
- `<section aria-label="...">` for major page sections â€” the label becomes a content anchor for AI summarization
- Heading hierarchy: `h1` â†’ `h2` â†’ `h3`, no skips. One `h1` per page.
- Every `<img>` needs descriptive `alt` text â€” not filename-style (`IMG_1234`), but descriptive (`Taipei city skyline at night from Xiangshan`)
- Dates use `<time datetime="ISO-8601">` format so AI agents can parse them correctly

### 4c. Structured Data Accuracy for LLM Knowledge Graphs

The JSON-LD `Person` schema in the root layout is the primary signal for AI agents building an identity profile for Max. Rules:

1. **Keep `sameAs` array current.** Every new public profile (Behance, Dev.to, etc.) should be added.
2. **Keep `knowsAbout` specific.** List technologies by name (e.g., "BigQuery" not "databases"). AI agents use this to match Max to technical queries.
3. **`hasCredential` must match the official certification name** exactly as issued by Google Cloud.
4. **`worksFor` should reflect current employer.** Update when Max changes jobs.

### 4d. Answer Engine Optimization (AEO)

AI search often answers questions by synthesizing the first substantive paragraph of a page. To make Max's content AEO-friendly:

| Content type | Rule |
|-------------|------|
| Blog posts | First paragraph = standalone summary of the key insight. Assume a reader will only see this paragraph in an AI-generated answer. |
| Portfolio projects | `shortDescription` (Sanity field) = 1 crisp sentence that could stand alone in an AI answer. No "this project..." preamble. |
| Photography captions | Descriptive and location-specific. "Street scene, Shibuya, Tokyo, 2023" beats "moments in motion". |
| About page | Open with "who I am + what I do" â€” no poetic opener. AI agents need the facts in the first sentence. |

### 4e. Canonical Author Identity

AI tools build author profiles from cross-web citations. Consistency matters:

- All external profiles (GitHub, LinkedIn, Behance, any future profiles) should reference `https://maxcsh.vercel.app` as the canonical URL.
- Blog posts and portfolio entries consistently use author name: **"SIH-HAN (Max) CHEN"** â€” this exact string in all JSON-LD `author.name` fields.
- The `Person` schema `sameAs` array is the authoritative list of Max's online presence.
